## 개요

### 데드라인

- c.f) 타임아웃
  - 개요
    - 클라이언트 애플리케이션이 RPC가 완료될 때까지 에러로 종료되기 전 얼마의 시간 동안 기다릴지를 지정
      - 기간(duration)으로 지정
      - 각 클라이언트에 개별로 적용
  - 단점
    - 요청 전체 수명주기에 타임아웃을 직접 적용 불가능
- 개요
  - 요청 시작 기준으로 절대적인 특정 시간으로 표현되며, 여러 서비스 호출에 걸쳐 적용됨
  - 애플리케이션이 데드라인을 설정하면 전체 요청 체인은 데드라인까지 응답해야 함
- 특징
  - 항상 데드라인을 지정하는것이 바람직
    - 데드라인을 지정하지 않으면 무한정 기다림
